# FEW-SHOT PROMPTING
# Provide 3-5 examples showing desired input → output mapping

SYSTEM PROMPT:
"""
You are a support ticket classifier. Analyze tickets and respond with JSON.
"""

EXAMPLES (few-shot):

Example 1:
User: "Me cobraron dos veces este mes"
Assistant: {
  "categoria": "facturacion",
  "prioridad": "critica",
  "sentimiento": "enojado"
}

Example 2:
User: "Tienen API para integrar?"
Assistant: {
  "categoria": "consulta_funcionalidad",
  "prioridad": "media",
  "sentimiento": "neutral"
}

Example 3:
User: "No puedo entrar, error 403"
Assistant: {
  "categoria": "cuenta",
  "prioridad": "alta",
  "sentimiento": "frustrado"
}

USER INPUT (new):
"Quiero cancelar mi suscripción, muy caro"

EXPECTED BEHAVIOR:
- Model learns the pattern from examples
- Mimics the structure and reasoning shown
- Consistent output format

PROS:
✓ Consistent output format
✓ Works with smaller models (gpt-4o-mini, gpt-3.5)
✓ Domain-specific behavior without fine-tuning
✓ Easy to iterate (just edit examples)

CONS:
✗ More tokens per request (examples add ~100-200 tokens)
✗ Need to carefully curate examples
